# Analyse SQL - Supermarché

Projet réalisé dans un cadre scolaire dans le module 
"Analyse de données avec SQL".

## Contexte
Analyse des données commerciales d'un supermarché à partir 
d'une base de données relationnelle composée de plusieurs tables :
clients, produits, ventes, employés, villes et pays.

## Objectifs
Répondre à des problématiques métier posées par différents 
services de l'entreprise.

## Requêtes réalisées
- **Q1** : Filtrage de produits pour une opération promotionnelle
- **Q2** : Analyse des ventes par produit
- **Q3** : Classement des vendeurs par chiffre d'affaires
- **Q4** : Produits les plus vendus par pays
- **Q5** : Analyse croisée client / vendeur / ville / pays

## Compétences SQL démontrées
- Filtres et conditions (WHERE, HAVING)
- Jointures multiples (JOIN)
- Agrégats (SUM, COUNT)
- Tri et regroupement (GROUP BY, ORDER BY)

## Outil utilisé
- DBeaver
